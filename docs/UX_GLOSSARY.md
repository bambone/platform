# UX glossary (интерфейс админки)

Единый словарь терминов для **Platform Console** и **Кабинета клиента**. Человекочитаемые подписи ролей в коде: [`App\Filament\Support\RoleLabels`](../app/Filament/Support/RoleLabels.php). Статусы доменов (подписи и подсказки): [`TenantDomainStatusCopy`](../app/Filament/Support/TenantDomainStatusCopy.php). Реестр настроек платформы: [`PlatformSettingRegistry`](../app/Filament/Support/PlatformSettingRegistry.php).

## Продукт и панели

| Системное / внутреннее | Термин в UI |
|------------------------|-------------|
| Tenant (клиент платформы) | **Клиент** (при первом упоминании можно: «Клиент — отдельный сайт и данные») |
| Platform panel | **Консоль платформы** |
| Tenant / Admin panel | **Кабинет клиента** |
| Public site | **Сайт для посетителей** |
| Slug | **URL-идентификатор** (с примером в helper) |
| Internal setting keys | В UI не показывать как основной текст; использовать человекочитаемые названия из реестра настроек |

## Роли платформы (Platform Console)

Системное имя хранится в БД; в интерфейсе **никогда** не показывать голый `snake_case`.

| Системное имя | Подпись в UI | Кратко о доступе |
|----------------|---------------|------------------|
| `platform_owner` | Владелец платформы | Полный доступ к консоли и критичным настройкам |
| `platform_admin` | Администратор платформы | Управление клиентами, тарифами, шаблонами |
| `support_manager` | Специалист поддержки | Работа с клиентами и диагностика (по политикам проекта) |

## Роли в кабинете клиента (membership)

| Системное имя | Подпись в UI |
|----------------|---------------|
| `tenant_owner` | Владелец клиента |
| `tenant_admin` | Администратор клиента |
| `booking_manager` | Менеджер бронирований |
| `fleet_manager` | Менеджер парка |

## Каталог и парк

| Концепция | Термин в UI |
|-----------|-------------|
| Motorcycle как карточка в каталоге | **Карточка в каталоге** (или «Модель в каталоге» — одна формулировка на проект) |

## Статусы (ориентиры для подписей)

Подписи в таблицах и формах — на русском; цвет badge — по смыслу (успех / внимание / ошибка / нейтрально).

- **Клиент (tenant):** Пробный, Активен, Приостановлен, В архиве (+ пояснение в helper при необходимости).
- **Заявка (lead):** см. `Lead::statuses()`.
- **Страница:** Черновик, Опубликован, Скрыт.
- **Домен — проверка:** задаётся в `TenantDomainStatusCopy` (подсказки «что делать дальше»).
- **Домен — SSL:** задаётся в `TenantDomainStatusCopy`.
- **Интеграция:** по типу записи; включена / выключена — тоггл с понятной подписью.
- **Единица парка:** Активна, Неактивна, На обслуживании.
- **Пользователь (учётная запись):** Активен, Заблокирован, Приглашён.

## Правила отображения

1. **Внутренний числовой `id`** не выводить в пользовательских таблицах как основную колонку, если нет бизнес-нужды.
2. **Raw JSON и key-value** — только в секции «Дополнительно» / «Для разработчика» с предупреждением.
3. **Роли и статусы** — только человекочитаемые строки из этого файла или из PHP-карт, согласованных с ним.
